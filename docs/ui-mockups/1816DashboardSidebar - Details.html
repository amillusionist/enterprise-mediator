Component Name

Description

Responsive, collapsible sidebar navigation drawer acting as the primary navigation structure for the application. Supports role-based menu items, collapsible states for desktop, and off-canvas behavior for mobile devices.

Design System Alignment

Implements the 'navigation.sidebar' specification with 240px/64px widths, primary-50 active states, and integrates the UserAvatarMenu molecule.

--font-secondary

Responsive Specifications

Mobile Adaptations

  • Sidebar transforms to off-canvas drawer (fixed position, z-index high)
  • Hidden by default (transform: translateX(-100%))
  • Full width backdrop overlay added when open
  • Close button appears in header

Tablet Adaptations

  • Can default to collapsed state (64px) to save screen real estate
  • Overlay behavior available if space is constrained

Desktop Enhancements

  • Collapsible toggle (240px <-> 64px)
  • Tooltip hints on hover when collapsed
  • Sticky positioning relative to viewport

Accessibility Features

Keyboard Support

Logical tab order through navigation items. Skip links supported (external to component).

Screen Reader Support

nav element with aria-label='Main Navigation'. Collapsed state uses aria-label on icon-only buttons.

Focus Management

Visible focus outlines. Toggle button indicates state via aria-expanded.

Aria Attributes

  • aria-current='page'
  • aria-expanded
  • aria-controls
  • aria-hidden

Color Contrast

Nav items pass AA standards (4.5:1) in both default and active states.

Usage Guidelines

When To Use

Use as the root layout component for all authenticated views (Admin, Finance, Vendor, Client).

Content Guidelines

Labels should be short (1-2 words). Icons are mandatory for collapsed state support.

Placement Rules

Placed at the root of the page structure, typically adjacent to a

content area.

Customization Options

  • .sidebar--collapsed: Reduces width to 64px
  • .sidebar--mobile-open: Slides sidebar in on mobile
  • Data-role attribute: Can be used to theme the sidebar header for different portals

Testing Recommendations

Browser Testing

Check transition smoothness on Safari and Chrome.

Device Testing

Verify touch targets on mobile drawer. Test open/close swipe gestures (if JS added).

Accessibility Testing

  • Verify tab navigation flow in collapsed state
  • Check aria-current behavior on active links
  • Ensure color contrast of active state background vs text

Performance Testing

Layout shift checks during collapse/expand animations.
