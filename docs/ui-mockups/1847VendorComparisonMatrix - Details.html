Description

A complex data comparison table designed for evaluating multiple vendors side-by-side against key metrics like cost, match score, and timeline. Features sticky headers for navigation efficiency on large datasets.

Design System Alignment

Utilizes core typography, color palette, and spacing tokens. Integrates atomic buttons and molecular match score indicators.

Responsive Specifications

Mobile Adaptations

  • Horizontal scrolling container enables viewing multiple columns on narrow screens
  • Sticky first column ensures context (labels) is never lost while scrolling
  • Reduced padding on cells to maximize viewport density

Tablet Adaptations

  • Maintain sticky headers
  • Ensure touch scrolling is smooth (-webkit-overflow-scrolling: touch)

Desktop Enhancements

  • Hover effects on rows to aid readability across wide tables
  • Visual highlighting of the 'Recommended' vendor column

Accessibility Features

Keyboard Support

Container is focusable and scrollable via arrow keys

Screen Reader Support

Uses scope='row' and scope='col' to associate data cells with headers properly

Focus Management

Interactive elements within cells (buttons) maintain natural tab order

Aria Attributes

  • role='grid'
  • aria-label='Vendor Comparison Matrix'

Color Contrast

Text and borders meet AA standards

Implementation Notes

Embedded Css Architecture

Single-file BEM architecture with scoped CSS variables for matrix dimensions

Component Organization

Table-based structure chosen over Grid for better semantic relationships and native sticky header support

Responsive Design

Relies on CSS 'position: sticky' which degrades gracefully in older browsers (though not targeted)

Accessibility Compliance

Semantic table markup ensures screen readers can navigate the 2D data structure effectively

Browser Support

Works in all modern browsers supporting position: sticky

Usage Guidelines

When To Use

When comparing 2-5 vendors across 3+ complex data points.

Content Guidelines

Keep row headers concise. Data cells should handle variable content heights gracefully.

Placement Rules

Place in the main content area of the Project Workspace.

Customization Options

  • Add .v-matrix--loading to parent to trigger skeleton states
  • Use .v-matrix--recommended on the table to highlight the first data column

Testing Recommendations

Browser Testing

Verify sticky behavior on Safari (iOS/Desktop) and Chrome

Device Testing

Test horizontal scroll interactions on touch devices

Accessibility Testing

  • Verify screen reader announces column header when navigating horizontally
  • Verify row header is announced when navigating vertically

Performance Testing

Check scroll performance with 10+ columns
