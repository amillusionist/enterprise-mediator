# 1 Overview

## 1.1 Diagram Id

SEQ-FF-001

## 1.2 Name

Semantic Vendor Matching

## 1.3 Description

After an Admin approves a Project Brief, they trigger the semantic search feature. The system uses the vector embeddings from the Project Brief's skills to query the database and find a ranked list of vendors with the most similar expertise.

## 1.4 Type

🔹 FeatureFlow

## 1.5 Purpose

To leverage AI-generated data to provide intelligent, data-driven recommendations for suitable vendors, fulfilling REQ-FUN-002's core matchmaking capability.

## 1.6 Complexity

High

## 1.7 Priority

🔴 High

## 1.8 Frequency

OnDemand

## 1.9 Participants

- REPO-FRONTEND-SPA
- REPO-API-MONOLITH
- REPO-DB-POSTGRES

## 1.10 Key Interactions

- Admin clicks 'Find Matching Vendors' on an approved Project Brief page.
- Frontend calls the API with the Project ID.
- API retrieves the vector embeddings for the skills associated with the project.
- API constructs and executes a vector similarity search query (e.g., cosine distance) against the vendor skills table in PostgreSQL using the pgvector extension.
- The database's HNSW index efficiently finds the nearest neighbors and returns a ranked list of vendors with a similarity score for each.
- API formats the results and returns the ranked list of recommended vendors to the frontend.

## 1.11 Triggers

- An Admin requests vendor recommendations for a specific, approved project brief.

## 1.12 Outcomes

- A ranked list of the most suitable vendors, along with their similarity scores, is displayed to the Admin for consideration.

## 1.13 Business Rules

- Only vendors with a status of 'Active' can be included in search results (REQ-BUS-001).
- The search algorithm and number of results should be configurable system parameters.

## 1.14 Error Scenarios

- No vendors in the database have skills that meet a minimum similarity threshold.
- The pgvector extension is not enabled or fails to execute the query, returning a server error.

## 1.15 Integration Points

- PostgreSQL with the pgvector extension and a pre-built HNSW index for performance.

## 2.3 Description

Technical sequence for an Administrator triggering a semantic search for vendors. This flow starts with a user interaction in the Next.js SPA, which calls a secured ASP.NET Core API endpoint. The backend orchestrates retrieving project skill vector embeddings and executes a high-performance vector similarity search against the PostgreSQL database using the pgvector extension and a pre-built HNSW index. The sequence details the internal application layers (Controller, Service, Repository) and the exact database queries involved, including error handling and performance considerations.
