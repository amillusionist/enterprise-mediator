# 1 Overview

## 1.1 Diagram Id

SEQ-UJ-004

## 1.2 Name

Vendor Asks Question on Project Brief and Admin Responds

## 1.3 Description

A vendor who has been invited to propose on a project uses the proposal portal's Q&A feature to ask a clarifying question. The System Admin is notified, posts an answer, and the answer becomes visible to all vendors invited to that project.

## 1.4 Type

🔹 UserJourney

## 1.5 Purpose

To facilitate a fair and transparent proposal process by allowing vendors to clarify requirements and ensuring all participants have the same information, as per REQ-FUN-003.

## 1.6 Complexity

Medium

## 1.7 Priority

🟡 Medium

## 1.8 Frequency

OnDemand

## 1.9 Participants

- REPO-FRONTEND-SPA
- REPO-API-MONOLITH
- REPO-DB-POSTGRES
- REPO-EXT-SES

## 1.10 Key Interactions

- Vendor views the Project Brief in the proposal portal and submits a question.
- Frontend sends the question to the API, which stores it with the project and an 'Unanswered' status.
- API notifies the System Admin of the new question.
- Admin logs in, views the question, and submits an answer.
- API saves the answer and updates the question's status to 'Answered'.
- API sends a notification to all vendors invited to the project that a new Q&A has been posted.

## 1.11 Triggers

- A vendor submits a question via the proposal submission portal.

## 1.12 Outcomes

- The vendor's question is answered.
- The answer is shared with all potential proposers for that project.
- A clear record of project clarifications is maintained.

## 1.13 Business Rules

- Vendor questions are submitted anonymously to the Admin.
- Admin answers are visible to all vendors for that specific project.

## 1.14 Error Scenarios

- Admin is not notified of a new question, causing delays.

## 1.15 Integration Points

- AWS SES for notifications.

# 2.0 Details

## 2.1 Diagram Id

SEQ-UJ-004

## 2.2 Name

Implementation: Vendor Q&A Workflow

## 2.3 Description

A comprehensive sequence detailing the two-part user journey for the project Q&A process. It begins with an invited Vendor submitting a question via the secure proposal portal. The system then processes this, notifies the System Administrator, who in turn logs in, provides an answer, which is then broadcast to all vendors invited to the project. The diagram specifies all UI state changes, API contracts, database transactions, and notification dispatches.
