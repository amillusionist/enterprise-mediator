# 1 Overview

## 1.1 Diagram Id

SEQ-IF-002

## 1.2 Name

Vendor Payout via Wise on Project Completion

## 1.3 Description

When a project is marked as 'Completed' and approved by a Finance Manager, the system initiates a payout to the vendor. It interacts with the Wise API to execute the international fund transfer to the vendor's registered account.

## 1.4 Type

🔹 IntegrationFlow

## 1.5 Purpose

To automate the final step of the financial lifecycle by securely and efficiently distributing funds to vendors, as per REQ-FUN-004 and REQ-TEC-001.

## 1.6 Complexity

High

## 1.7 Priority

🚨 Critical

## 1.8 Frequency

OnDemand

## 1.9 Participants

- REPO-API-MONOLITH
- REPO-DB-POSTGRES
- REPO-EXT-WISE
- REPO-EXT-SES

## 1.10 Key Interactions

- An Admin marks the project as 'Completed'.
- This triggers a payout request that enters a 'Pending Approval' state.
- A Finance Manager reviews the payout details and clicks 'Approve Payout'.
- API retrieves the vendor's payment details and the approved payout amount.
- API calls the Wise API to create and execute the fund transfer, handling currency conversion if necessary.
- Wise processes the payout and provides a transaction status via its API.
- API records the payout transaction details in the internal ledger.
- API sends a payout confirmation notification to the vendor via SES.

## 1.11 Triggers

- A Finance Manager approves a pending payout for a completed project.

## 1.12 Outcomes

- Funds are securely transferred from the company's Wise account to the vendor's account.
- The payout is recorded in the system's financial records with exchange rates and fees.
- The vendor is notified of the completed payment.

## 1.13 Business Rules

- A payout cannot be initiated until corresponding client funds are successfully held in escrow (REQ-BUS-002).
- All payouts must be approved by a user with the Finance Manager role.
- The payout amount cannot exceed the accepted proposal amount without an explicit override and justification.

## 1.14 Error Scenarios

- The vendor's registered payment details are invalid, causing the Wise transfer to fail.
- The Wise API call fails due to transient network issues, requiring retries.
- Insufficient funds in the source account to cover the payout.

## 1.15 Integration Points

- Wise API for international payouts.
