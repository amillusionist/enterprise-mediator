# 1 Overview

## 1.1 Diagram Id

SEQ-UJ-001

## 1.2 Name

Admin Invites a New User

## 1.3 Description

An authenticated System Administrator invites a new user (Internal, Client, or Vendor) to the platform by providing their email address and assigned role. The system validates the request, generates a unique, single-use, time-limited registration token, and dispatches an invitation email to the specified address.

## 1.4 Type

🔹 UserJourney

## 1.5 Purpose

To securely onboard new users onto the platform as per REQ-FUN-001, initiating the user lifecycle through a controlled, admin-driven invitation process.

## 1.6 Complexity

Medium

## 1.7 Priority

🔴 High

## 1.8 Frequency

OnDemand

## 1.9 Participants

- REPO-FRONTEND-SPA
- REPO-API-MONOLITH
- REPO-DB-POSTGRES
- REPO-EXT-SES

## 1.10 Key Interactions

- Admin submits the new user's email and role via the UI's user management panel.
- API validates the Admin's permissions (must be SystemAdministrator role).
- API checks if the email address is already in use.
- API generates a cryptographically secure, unique, and time-limited token.
- API creates a new user record with a 'Pending' status and stores a hash of the token in the database.
- API calls the notification service to send a templated invitation email via AWS SES, including the registration link with the token.

## 1.11 Triggers

- System Administrator clicks 'Invite User' in the admin panel and submits the form.

## 1.12 Outcomes

- A new user record is created in a 'Pending' state, awaiting activation.
- An invitation email with a secure registration link is sent to the prospective user.

## 1.13 Business Rules

- Only System Administrators can invite new users (REQ-SEC-001).
- The invitation token must expire within a configurable period, e.g., 24 hours (REQ-FUN-001).

## 1.14 Error Scenarios

- The provided email address is already registered.
- The email sending service (SES) fails, and the API must handle the error gracefully.
- Admin lacks the required permissions to perform the action.

## 1.15 Integration Points

- AWS SES for sending transactional emails.

# 2.0 Details

## 2.1 Diagram Id

SEQ-UJ-002

## 2.2 Name

Client Approves a Milestone via Secure Email Link

## 2.3 Description

This sequence details the end-to-end user journey initiated when a Client Contact clicks a secure, time-limited link from an email to approve or reject a project milestone. The flow is designed for an unauthenticated user, relying on a single-use token (JWT) for authorization. It covers token validation, interaction with the dedicated approval UI, the atomic update of the milestone and token status, and the subsequent notification to internal administrators. The process emphasizes security, performance for a fast user experience, and robust error handling for invalid, expired, or previously used tokens.
