# 1 Overview

## 1.1 Diagram Id

SEQ-BP-002

## 1.2 Name

Admin Compares Proposals and Awards Project

## 1.3 Description

After vendors submit proposals for a project, the Admin uses a dedicated comparison dashboard to review them side-by-side. The Admin then selects one proposal to accept, which atomically transitions the project and all related proposal statuses.

## 1.4 Type

🔹 BusinessProcess

## 1.5 Purpose

To facilitate the crucial business decision of selecting a vendor for a project and formally awarding the work, triggering the financial workflow, as per REQ-FUN-003.

## 1.6 Complexity

Medium

## 1.7 Priority

🚨 Critical

## 1.8 Frequency

OnDemand

## 1.9 Participants

- REPO-FRONTEND-SPA
- REPO-API-MONOLITH
- REPO-DB-POSTGRES
- REPO-EXT-SES

## 1.10 Key Interactions

- Admin navigates to the project's proposal comparison dashboard.
- Frontend requests and displays all submitted proposals with key metrics (Cost, Timeline, Vendor Rating) in a standardized comparison view.
- Admin selects a winning proposal and clicks the 'Award Project' button.
- API receives the request and, within a single database transaction, performs the following:
- 1. Updates the accepted proposal's status to 'Accepted'.
- 2. Updates the statuses of all other proposals for that project to 'Rejected'.
- 3. Updates the project's status to 'Awarded' and links the awarded Vendor ID.
- 4. Logs the state transitions in the audit trail.
- After the transaction commits, the API sends notifications to the accepted and rejected vendors via AWS SES.

## 1.11 Triggers

- An Admin decides to award a project after reviewing submitted proposals.

## 1.12 Outcomes

- The project is officially awarded to a specific vendor.
- The project status is updated to 'Awarded', unlocking the invoicing step.
- All involved vendors are automatically notified of the decision.
- A comprehensive audit trail entry is created for the decision.

## 1.13 Business Rules

- A project status can only be moved to 'Awarded' after a proposal has been formally 'Accepted' (REQ-BUS-001).
- Only one proposal can be in the 'Accepted' state per project.

## 1.14 Error Scenarios

- Admin attempts to accept a proposal for a project that is not in the 'Proposed' state.
- The database transaction fails, rolling back all status changes.

## 1.15 Integration Points

- AWS SES for sending notifications to vendors.
